Laboratorio 6 - Programación orientada a eventos

Preguntas

¿Por qué es una buena práctica usar JavaScript para checar que sean válidos los inputs de las formas antes de enviar los datos al servidor?
Es buena practica y que favorece al UX porque el usuario no tiene que esperar a que se envie la informacion al servidor para saber si los datos son validos o no, ademas de que se pueden mostrar mensajes personalizados para cada error. Esto reduce mucho los errores del usuario y mejora la experiencia de usuario.

Fuente

¿Cómo puedes saltarte la seguridad de validaciones hechas con JavaScript?
"es tan sencillo como introducir un par de palabras reservadas en el código HTML utilizando la consola del desarrollador de nuestro navegador."
Puedes saltarte la seguridad de validaciones hechas con JavaScript deshabilitando el JavaScript en el navegador o manipulando el DOM con herramientas de desarrollo del navegador. Como por ejemplo, puedes eliminar los atributos de validación de los campos del formulario o modificar el código JavaScript para que no realice las validaciones. Es por esto que las validaciones siempre deben ser complementadas con validaciones en el backend para hacer que tu aplicación sea más segura.

Fuente

Si te puedes saltar la seguridad de las validaciones de JavaScript, entonces ¿por qué la primera pregunta dice que es una buena práctica?
Porque las validaciones en el front hacen que la experiencia de usuario sea mejor, ya que muchas veces puede ser molesto para el usuario tener que esperar a que se envie la informacion al servidor para saber si los datos son validos o no, ademas de que se puede mostrar el porque de cada error. Por otro lado, validarlo con el backend no esta opuesto a validarlo con el frontend, sino que se complementan para hacer que tu aplicación sea más segura y con buen UX. Es decir, las validaciones en el frontend son para mejorar la experiencia de usuario, mientras que las validaciones en el backend son para mejorar la seguridad de la aplicación.
